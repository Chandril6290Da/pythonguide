Syntax & Basics

Python is a dynamically typed language — no type declarations, no semicolons, no curly braces. Indentation is the structure.

Variables & Data Types

python 
# Five core built-in types
age      = 25           # int
price    = 9.99         # float
name     = "Alice"      # str
is_admin = True         # bool
nothing  = None         # NoneType

# Multiple assignment
a, b, c = 1, 2, 3
x = y = z = 0
a, b = b, a              # swap!

# Big numbers with underscores
population = 1_000_000_000

int42, -7, 1_000❌Unlimited size integers
float3.14, 1.5e3❌64-bit decimal numbers
str"hello"❌Sequence of characters

Type Casting

python 
int("42")       # 42
float("3.14")   # 3.14
str(100)        # "100"
bool(0)         # False  (0, "", [], {}, None are falsy)
bool("hi")      # True

f-Strings & Print

python 
name, score = "Alice", 98.5
print(f"Name: {name}")          # Name: Alice
print(f"Score: {score:.1f}")     # Score: 98.5
print(f"Big: {1_000_000:,}")     # Big: 1,000,000
print("a", "b", sep="-", end="!\n") # a-b!

Indentation is mandatoryPython uses 4 spaces per level instead of curly braces. Missing or wrong indentation causes IndentationError.

Operators

Python has 7 categories of operators. Understanding precedence and short-circuit evaluation is essential.

Arithmetic Operators

python 
a, b = 10, 3

a + b    # 13   addition
a - b    # 7    subtraction
a * b    # 30   multiplication
a / b    # 3.333...  always returns float!
a // b   # 3    floor division (truncates toward -infinity)
a % b    # 1    modulo (remainder)
a ** b   # 1000 exponentiation (10^3)

# Modulo tricks
10 % 2 == 0   # even check
9 ** 0.5      # 3.0  square root!

Comparison & Logical

python 
# Chained comparison - Python superpower!
0 < x < 10         # True if x between 0 and 10
18 <= age <= 65    # working age range

# Short-circuit evaluation
False and print("skipped")  # right side never runs!
True  or  print("skipped")  # right side never runs!

# or returns first truthy value
name = ""
display = name or "Anonymous"  # "Anonymous"

# Identity vs equality
a == b    # same VALUE
a is b    # same OBJECT in memory
x is None # correct way to check for None

03 📝

Strings

Strings are immutable sequences of characters. Python has 50+ string methods built in.

Indexing & Slicing

python 
word = "Python"
#       0 1 2 3 4 5   positive indices
#      -6-5-4-3-2-1   negative indices

word[0]     # "P"
word[-1]    # "n"  last char
word[1:4]   # "yth"
word[::-1]  # "nohtyP"  reversed!
word[:3]    # "Pyt"
word[2:]    # "thon"

Essential String Methods

python 
s = "  Hello, Python World!  "

# Case methods
s.upper()         # "  HELLO, PYTHON WORLD!  "
s.lower()         # "  hello, python world!  "
s.title()         # "  Hello, Python World!  "
s.swapcase()      # flip all cases

# Strip whitespace or chars
s.strip()         # "Hello, Python World!"
s.strip("! ")    # remove specific chars from ends

# Search methods
s.find("Python")        # 9 (index), -1 if missing
s.count("l")            # 3
s.startswith("  Hello")  # True
s.endswith("  ")         # True

# Replace, split, join
s.replace("Python", "Java")   # replaces substring
"a,b,c".split(",")            # ["a","b","c"]
" ".join(["Hello", "World"])  # "Hello World"

# Type checks
"hello".isalpha()    # True
"12345".isdigit()    # True
"abc123".isalnum()   # True
💡
Strings are immutableEvery string method returns a new string. The original is never modified. To "change" a string, reassign the variable.
04 🔒

Tuples

Tuples are ordered, immutable sequences. Faster than lists, hashable, and perfect for fixed data.

Creating & Unpacking

python 
# Creating tuples
point  = (10, 20)
single = (42,)        # trailing comma required for single!
packed = 1, 2, 3      # no parens needed - still a tuple

# Unpacking
x, y = point            # x=10, y=20
a, b = b, a             # swap variables!

# Extended unpacking with *
first, *rest = (1, 2, 3, 4, 5)
# first=1, rest=[2, 3, 4, 5]

first, *middle, last = (1, 2, 3, 4, 5)
# first=1, middle=[2,3,4], last=5

# Named tuple - named access!
from collections import namedtuple
Person = namedtuple("Person", ["name", "age", "city"])
alice = Person("Alice", 30, "NY")
print(alice.name)   # Alice
✅ Use Tuples For

Fixed data like coordinates, RGB values, DB records, function return values

🔑 Dict Keys

Tuples are hashable so they can be dictionary keys — lists cannot

⚡ Performance

Tuples are ~3x faster to create than lists and use less memory

05 🔵

Sets

Sets are unordered collections of unique elements. O(1) membership lookup and powerful mathematical set operations.

Creating & Modifying

python 
fruits = {"apple", "banana", "cherry"}
empty  = set()   # NOT {} -- that creates a dict!

# Duplicates auto-removed
nums = {1, 2, 2, 3, 3, 3}   # {1, 2, 3}

# Add / Remove
fruits.add("mango")      # add single
fruits.update(["kiwi", "grape"]) # add multiple
fruits.discard("banana") # remove safely (no error)
fruits.remove("apple")   # remove (KeyError if missing)

Set Operations

python 
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b   # {1,2,3,4,5,6}  Union (all elements)
a & b   # {3, 4}          Intersection (common)
a - b   # {1, 2}          Difference (in a not b)
a ^ b   # {1,2,5,6}       Symmetric diff (not common)

# Relationship checks
a.issubset(b)    # a <= b
a.issuperset(b)  # a >= b
a.isdisjoint(b)  # no common elements?

# Remove duplicates from list
dupes  = [1, 2, 2, 3, 3, 3]
unique = list(set(dupes))   # [1, 2, 3]
ℹ️
Empty set gotcha{} creates an empty dict, not a set! Use set() for an empty set. This is a very common mistake.
06 📚

Dictionaries

Dictionaries are ordered, mutable key-value stores with O(1) lookup by key. The most versatile Python data structure.

Creating & Accessing

python 
person = {"name": "Alice", "age": 30, "city": "NY"}

# Access
person["name"]                # "Alice" - KeyError if missing!
person.get("phone", "N/A")   # "N/A" - safe access

# Add / Update
person["email"] = "a@b.com"
person.update({"age": 31, "job": "Engineer"})
person.setdefault("country", "USA")  # set only if missing

# Remove
age  = person.pop("age")      # remove and return value
last = person.popitem()        # remove last inserted item
del person["city"]            # delete by key

# Iterate - most common
for k, v in person.items():
    print(f"{k}: {v}")

Dict Comprehension & Merge

python 
# Comprehensions
squares  = {x: x**2 for x in range(1, 6)}
filtered = {k: v for k, v in scores.items() if v >= 60}
swapped  = {v: k for k, v in original.items()}

# Merge dicts
merged = {**dict1, **dict2}    # Python 3.5+
merged = dict1 | dict2         # Python 3.9+

# defaultdict - auto creates default value
from collections import defaultdict
counter = defaultdict(int)
for word in words:
    counter[word] += 1   # no KeyError ever!

Control Flow

Control flow determines the order code executes — decisions, loops, and jump statements.

Conditionals

python 
score = 78

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"

# Ternary - one-line if/else
status = "Adult" if age >= 18 else "Minor"

# match statement (Python 3.10+)
match command:
    case "quit":
        quit()
    case "help" | "?":
        show_help()
    case _:
        print("Unknown")

Loops & Loop Control

python 
# for loop - iterate sequences
for i in range(5): ...              # 0 1 2 3 4
for i in range(0, 20, 5): ...      # 0 5 10 15
for i, v in enumerate(lst, 1): ... # index + value
for a, b in zip(list1, list2): ...  # parallel iteration

# while loop - condition based
while count < 10:
    count += 1

# Loop control keywords
break       # exit loop immediately
continue    # skip to next iteration
pass        # do nothing (placeholder)

# for...else - Python superpower!
for user in users:
    if user == target:
        print("Found!")
        break
else:
    print("Not found")  # runs only if no break!
💡
for vs whileUse for when you know the iterations or are iterating a sequence. Use while when waiting for a condition to change or when you don't know how many iterations you need.

Functions

Functions are first-class objects in Python — they can be assigned, passed, and returned like any value.

Arguments — All Types

python 
def create_user(username, password, *roles,
                active=True, **metadata):
    print(username, roles, active, metadata)

create_user("alice", "pass", "admin", "editor",
            active=True, email="a@b.com")

# *args  collects extra positional args into a tuple
# **kwargs collects extra keyword args into a dict

# Return multiple values (returns a tuple)
def min_max(nums):
    return min(nums), max(nums)

low, high = min_max([3, 1, 9, 4])

Lambda, Closures & Decorators

python 
# Lambda - anonymous one-line function
square   = lambda x: x ** 2
students.sort(key=lambda s: s["age"])

# Closure - function that remembers enclosing scope
def make_greeting(greeting):
    def greet(name):
        return f"{greeting}, {name}!"
    return greet

hello = make_greeting("Hello")
hello("Alice")   # "Hello, Alice!" - closure!

# Decorator - wrap a function to add behavior
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        import time
        start = time.time()
        result = func(*args, **kwargs)
        print(f"Took {time.time()-start:.4f}s")
        return result
    return wrapper

@timer
def slow_func():
    import time; time.sleep(1)

File I/O

File I/O lets your program read and write data permanently. Always use with — it guarantees the file closes even if an error occurs.

Reading & Writing

python 
# Reading modes
with open("file.txt", "r", encoding="utf-8") as f:
    content = f.read()          # entire file as string
    lines   = f.readlines()     # list of all lines
    line    = f.readline()      # one line at a time
    for line in f: ...          # memory-efficient iteration!

# Writing (overwrites existing content!)
with open("file.txt", "w", encoding="utf-8") as f:
    f.write("Hello\n")
    f.writelines(["line1\n", "line2\n"])

# Append (never overwrites)
with open("log.txt", "a", encoding="utf-8") as f:
    f.write("New log entry\n")

# Error handling for files
try:
    with open("missing.txt") as f:
        data = f.read()
except FileNotFoundError:
    print("File not found!")
except PermissionError:
    print("No permission!")

CSV & JSON

python 
import csv, json

# CSV - DictReader gives a dict per row
with open("data.csv", "r", encoding="utf-8") as f:
    for row in csv.DictReader(f):
        print(row["name"], row["age"])

with open("out.csv", "w", newline="") as f:
    w = csv.DictWriter(f, fieldnames=["name", "age"])
    w.writeheader()
    w.writerows(data)

# JSON - load from file / dump to file
with open("data.json") as f:
    data = json.load(f)          # file to dict

with open("out.json", "w") as f:
    json.dump(data, f, indent=4) # dict to file

# String conversions
json.loads('{"x": 1}')   # JSON string to dict
json.dumps({"x": 1})     # dict to JSON string
